
### 0.2 环境探测（只读）✅

```bash
ssh arm 'bash -s' <<'REMOTE'
lscpu | grep -iE 'Model name|Architecture|CPU\(s\)|BogoMIPS'
grep -iE 'implementer|part' /proc/cpuinfo | sort -u
systemd-detect-virt
cat /proc/sys/kernel/perf_event_paranoid
perf list hw
for i in 0 1 2 3; do cat /sys/devices/system/cpu/cpu0/cache/index$i/coherency_line_size; done
REMOTE
```
结果：鲲鹏 920 / TaiShan v110 / aarch64 / 4 核 2.6GHz / KVM guest / `paranoid=-1` / cache line L1·L2=64B·L3=128B。

### 0.3 PMU 事件方案 ✅（本次核心）

```bash
# 实测 4 关键事件
perf stat -e cache-misses,L1-dcache-load-misses,LLC-load-misses,dTLB-load-misses -- sleep 2
# → cache-misses/L1-dcache/dTLB 有数；LLC-load-misses <not supported>
# 发现 perf 4.19 不认 pmu/符号名/，破 raw code：
cat /sys/bus/event_source/devices/armv8_pmuv3_0/events/ll_cache_miss_rd   # → event=0x037
perf stat -e armv8_pmuv3_0/event=0x037,name=LLC-load-misses/ -- sleep 2   # → 非零 ✅
```
结论：2 个事件用 raw code 替代（`LLC-load-misses`=0x037、`branch-instructions`=0x021），其余通用名。详见 `知识点` 文档第三节。
