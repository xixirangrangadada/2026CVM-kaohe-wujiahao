### 0.2 环境探测（只读）✅

```bash
ssh arm 'bash -s' <<'REMOTE'
lscpu | grep -iE 'Model name|Architecture|CPU\(s\)|BogoMIPS'
```

### 0.3 PMU 事件方案 ✅（本次核心）

```bash
# 实测 4 关键事件
perf stat -e cache-misses,L1-dcache-load-misses,LLC-load-misses,dTLB-load-misses -- sleep 2
# → cache-misses/L1-dcache/dTLB 有数；LLC-load-misses <not supported>
# 发现 perf 4.19 不认 pmu/符号名/，破 raw code：
cat /sys/bus/event_source/devices/armv8_pmuv3_0/events/ll_cache_miss_rd   # → event=0x037
perf stat -e armv8_pmuv3_0/event=0x037,name=LLC-load-misses/ -- sleep 2   # → 非零 ✅
```

<details>
<summary>📋 实测原始输出（点开）</summary>

```
# 4 关键事件实测
Performance counter stats for 'sleep 2':
         1,028,113      cycles
           589,562      instructions              #    0.57  insn per cycle
             5,590      cache-misses
             5,590      L1-dcache-load-misses     ← 与 cache-misses 完全相同（perf 在 ARM 映射冗余）
   <not supported>      LLC-load-misses           ← 通用名失败
             1,570      dTLB-load-misses
       2.000943617 seconds time elapsed

# raw code 破解（绕过 perf 抽象层）
$ perf stat -e armv8_pmuv3_0/event=0x037,name=LLC-load-misses/ -- sleep 2
Performance counter stats for 'sleep 2':
             4,493      LLC-load-misses           ← raw code 成功 ✅
       2.000824626 seconds time elapsed

$ perf stat -e armv8_pmuv3_0/event=0x021,name=branch-instructions/ -- sleep 2
Performance counter stats for 'sleep 2':
           118,740      branch-instructions       ← raw code 成功 ✅
       2.000799256 seconds time elapsed
```
</details>

结论：2 个事件用 raw code 替代（`LLC-load-misses`=0x037、`branch-instructions`=0x021），其余通用名。详见 `知识点` 文档第三节。
